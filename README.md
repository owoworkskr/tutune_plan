# 기획 공간 (Planning Space)

서비스 아이디어를 발굴하고, PRD(제품 요구사항 문서)로 구체화하는 기획 작업 공간입니다.

## 📁 폴더 구조

```
.
├── README.md              # 이 문서 (사용 안내)
├── docs/
│   └── PRD-작성가이드.md   # PRD를 어떻게 쓰는지에 대한 가이드
├── templates/
│   └── PRD-템플릿.md       # 복사해서 바로 쓰는 PRD 빈 양식
└── ideas/
    ├── README.md          # 아이디어 목록/현황 보드
    └── 0000-예시-PRD.md    # 작성 예시 (참고용)
```

## 🚀 시작하기

1. **아이디어가 떠오르면** → `ideas/` 폴더에 새 파일을 만듭니다.
   - 파일명 규칙: `NNNN-서비스이름.md` (예: `0001-오늘의점심.md`)
   - `templates/PRD-템플릿.md`를 복사해서 채워 넣으세요.
2. **어떻게 쓸지 막히면** → `docs/PRD-작성가이드.md`를 읽어보세요.
3. **완성된 예시가 궁금하면** → `ideas/0000-예시-PRD.md`를 참고하세요.
4. **현황을 한눈에** → `ideas/README.md`의 아이디어 보드에 한 줄씩 추가합니다.

## 🔁 작업 흐름

```
아이디어 씨앗  →  한 줄 정리(ideas 보드)  →  PRD 초안(템플릿)  →  리뷰/보완  →  개발 착수
   (Seed)          (One-liner)              (Draft)           (Review)      (Build)
```

작게 시작해도 괜찮습니다. 문제 정의 한 줄부터 채우면 됩니다.
